## 입력 정보
[여기에 IA/메뉴구조도 정보를 붙여넣거나 파일 경로 제공]

---

## 생성 규칙

### 1. 문서 구조
- 제목: "{화면영역}_FO_{디바이스타입} 기능 정의서"
- 프로젝트명, 작성일(오늘 날짜), 작성자, 버전(v1.0) 자동 생성
- 버전 히스토리 테이블 포함

### 2. 각 화면/메뉴별로 다음 섹션 생성

#### 화면 정보
- **화면ID**: {대분류}_{중분류}_{화면번호} 형식 (예: FO_HOME_01)
- **화면명**: IA에서 정의된 메뉴명 사용

#### 주요기능
- 해당 화면의 핵심 기능 3-5개를 bullet point로 나열
- 사용자 관점에서 기술

#### 화면 설명
- 화면의 목적과 역할을 2-3문장으로 설명
- 사용자 시나리오 포함

#### 상세 기능 목록 (테이블)
| 번호 | 구분 | 기능명 | 기능 설명 | 비고 |
|------|------|--------|-----------|------|

**구분 카테고리**:
- 조회: 데이터 조회/검색 기능
- 입력: 데이터 입력/등록 기능
- 수정: 데이터 수정 기능
- 삭제: 데이터 삭제 기능
- 연동: 외부 시스템/API 연동
- 유효성: 입력값 검증 로직
- 권한: 접근 권한 제어
- 알림: 사용자 알림/피드백
- 네비게이션: 화면 이동/라우팅

**기능 설명 작성 가이드**:
- 명확하고 구체적으로 작성 (누가, 무엇을, 어떻게)
- 조건이 있는 경우 "~인 경우" 명시
- 결과/동작을 포함

### 3. 기능 추론 로직
IA 메뉴 정보에서 다음을 자동 추론:
- **목록형 화면**: 조회, 검색, 필터, 페이징, 정렬 기능
- **상세 화면**: 조회, 수정, 삭제, 이전/다음 이동 기능
- **등록/작성 화면**: 입력, 유효성 검증, 임시저장, 제출 기능
- **대시보드**: 통계 조회, 차트 표시, 기간 필터 기능
- **설정 화면**: 조회, 수정, 저장 기능

### 4. 출력 형식
- Markdown 파일로 생성
- 파일명: `{화면영역}_FunctionalSpecification_v1.0.md`
- UTF-8 인코딩
- 깔끔한 테이블 정렬

## 실행 지시사항

이제 위 규칙에 따라 제공된 IA/메뉴구조도를 기반으로 기능정의서를 생성해주세요.

**생성 시 주의사항:**
1. 모든 화면에 대해 일관된 형식 유지
2. 화면ID는 계층구조를 반영하여 자동 생성
3. 각 기능은 실제 구현 가능한 수준으로 구체적으로 작성
4. 비슷한 유형의 화면은 유사한 기능 구조 적용
5. 필수 기능(접근 권한, 오류 처리 등)은 모든 화면에 포함

## 사용 팁

### IA 데이터 입력 예시

**JSON 형식:**
```json
{
  "메인": {
    "홈": ["대시보드", "최근 활동"],
    "프로젝트": ["프로젝트 목록", "프로젝트 상세", "프로젝트 생성"]
  }
}
```

**표 형식:**
| 대분류 | 중분류 | 소분류 | 화면명 |
|--------|--------|--------|--------|
| 메인 | 홈 | - | 대시보드 |
| 메인 | 프로젝트 | 목록 | 프로젝트 목록 |

**계층구조 텍스트:**
```
1. 메인
  1.1 홈
    - 대시보드
  1.2 프로젝트
    - 프로젝트 목록
    - 프로젝트 상세
```

### 커스터마이징 옵션
- 특정 화면만 생성: `특정 메뉴만 생성: [메뉴명 리스트]`
- 구분 카테고리 추가: `추가 구분: [카테고리명]`
- 프로젝트 정보 지정: `프로젝트명: [이름], 작성자: [이름]`
